Dataset Overview

100,000Records
9Features
8.5%Diabetic Cases
0Missing Values

The dataset originates from the Kaggle Diabetes Prediction Dataset and contains demographic, behavioural, and clinical features such as age, BMI, blood glucose level, HbA1c level, smoking history, hypertension, and heart disease status. The binary target variable diabetes is highly imbalanced (≈ 91.5 % negative), which motivated our use of SMOTE during model training.

Exploratory Data Analysis

Feature Distributions

Histograms show that blood glucose and HbA1c levels are right-skewed, indicating a concentration of values at the lower end with a long tail toward higher readings — typical for a population where most individuals are not diabetic.

Outlier Detection — Box Plots

BMI and blood glucose level contain significant outliers. These were capped using the IQR method in the feature engineering phase to prevent them from distorting model training.

Gender vs Diabetes

The diabetic-to-non-diabetic ratio is broadly similar across genders, suggesting gender alone is a weak predictor.

Smoking History vs Diabetes

Current and former smokers show a slightly elevated rate of diabetes, consistent with clinical literature linking smoking to insulin resistance.

Correlation Heatmap

Blood glucose level (r ≈ 0.42) and HbA1c level (r ≈ 0.40) show the strongest correlations with diabetes, validating their clinical significance. BMI and age show moderate positive correlations.

🔍 Key Insights

  • Blood glucose level is the strongest single predictor of diabetes.
  • HbA1c level is a close second and highly clinically meaningful.
  • BMI contains significant outliers that required capping.
  • Age shows moderate positive correlation with diabetes onset.
  • Smoking history contributes a weak-to-moderate signal.
  • The dataset is clean with zero missing values.
Member 2 — Feature Engineering

Pre-processing Pipeline

01

Label Encoding

Gender encoded as Female=0, Male=1, Other=2 using LabelEncoder.

02

One-Hot Encoding

Smoking history (6 categories) expanded with pd.get_dummies to avoid ordinal assumptions.

03

IQR Outlier Capping

BMI and blood glucose clipped to [Q1 − 1.5·IQR, Q3 + 1.5·IQR] to limit leverage of extreme values.

04

Feature Selection

Retained only features with |correlation with target| > 0.05, reducing noise dimensionality.

SMOTE — Class Balance

The training set was severely imbalanced (≈ 8.5 % positive). SMOTE synthesised minority-class samples until both classes were equal, ensuring models did not simply predict "no diabetes" every time.

Member 3 — Machine Learning & Evaluation

Model Training & Comparison

Four classifiers were evaluated across three dataset versions (unscaled, standardised, normalised). Because misdiagnosing a diabetic patient as healthy is far more dangerous than the reverse, Recall was used as the primary selection criterion.

Accuracy · Recall · F1 Across All Configurations

Confusion Matrices — Best Model

Confusion matrices across all three dataset versions for the best-performing model, showing the trade-off between True Positives and False Negatives.

🏆 Best Model

Random Forest on the standardised dataset achieved the highest recall for the diabetic class while maintaining strong overall accuracy and F1-score — making it the recommended model for deployment.
